# 🛠️ Agent Requirements: Database Migration & Schema Update Workflow

This document outlines the **strict** requirements for the AI agent (Antigravity) when dealing with database schema changes in this project.

## 🗄️ Drizzle Industrial-Grade Practices

To maintain consistency between development, staging, and production environments, all database changes must follow the standard Drizzle migration flow:

1. **Schema Source of Truth**: All changes start in `src/lib/schema.ts`.
2. **Generated Migration Artifacts**: Every schema modification (adding/deleting tables, columns, constraints) **MUST** result in a generated `.sql` migration file using Drizzle Kit.
3. **Local Database Synchronization**: The local `local.db` (or any testing database) **MUST** be updated via the generated migrations, not by manual SQL execute or ad-hoc table pushes.

## 🛠️ Required Commands

When a schema change is implemented, the agent must run:

1. `npm run db:generate`: To create the SQL migration file in the `drizzle/` directory.
2. `npm run db:migrate`: To apply the generated SQL migration to the local `local.db`.

## 🚨 Prohibited Actions

- **NO ad-hoc field additions**: Do not write raw SQL (e.g., `ALTER TABLE...`) directly in your code to change the database structure.
- **NO manual schema drift**: Do not manually modify the SQLite database using external managers without reflecting it in the schema and generating a migration.
- **NO ignoring migration errors**: If a migration fails, diagnose and fix the schema or migration SQL before proceeding.
