# 需求文档 (Requirements)

## 基础技术与 UI

* 基于已有应用nextjs技术栈和 UI 前端实现基础。

## 部署与环境要求

1. **架构支持**
    * 应用必须支持 **本地环境 (Local)** 与 **远程环境 (Remote/Cloud)** 的部署机制和切换机制。
2. **数据库选型**
    * 本地环境使用 `sqlite` 作为数据库解决方案，以降低本地或 NAS 依赖的运维难度。
3. **容器化解决方案**
    * 需支持 Docker 完整容器化部署方案。
    * 特别强调：兼顾并支持群晖 (Synology) 等本地 NAS 系统的 Docker 管理平台（配置友好及挂载目录合理等）。
    * **数据持久化映射规则**：在 Docker 部署时，必须把如下目录单独映射出来，确保数据安全和迁移方便：
        * `database/` - SQLite 数据库文件目录
        * `uploads/images/` - 图片单独上传目录（内部按日期如 `YYYY-MM-DD` 自动形成子目录）
        * `uploads/voices/` - 语音文件独立目录
4. **网络与访问安全**
    * 提供域名的完整支持及相关解决方案设置说明。
    * 需要支持 **配置局域网证书方案 (SSL within LAN)**，能够实现局域网访问时的 HTTPS 加密（如通过内网解析、自签证书、通配符证书配置或其他反向代理方式）。
5. **支持web push**
    * 能够实现消息实时推送，在应用未激活时依然能接收到提醒。

## 功能与体验要求 (Features & Experience)

1. **极致的交互体验 (Smooth & Fluid UI)**
    * 深度集成 `motion.dev`，确保从页面切换到细节交互等每一个 UI 节点，都具备极致的丝滑度与流畅度 (Smooth & Fluid)。
2. **离线优先与数据同步 (PWA & Offline-First)**
    * 提供离线数据留存能力；在网络恢复连接时，自动将本地产生的数据无缝同步至服务器端（兼容本地 Local、NAS 或 Remote 远程部署形式）。

## 核心底层系统 (Core Foundation Systems)

1. **角色权限与多端协同**
    * **共用单端设计**：家长和孩子共用一套应用端。
    * **家长模式 (Parent Mode/Admin)**：支持由家长进入管理员模式，进行初始化配置、参数调整、功能控制以及手动奖惩。
    * **实时协同 (Real-time Interactive)**：家长和儿童的数据实时同步。例如，家长在自己设备上或家长模式下给予一次奖励，儿童端会立刻通过 Web Push 接收到动效或通知。

2. **双轨驱动系统 (Two-track System)**
    * **奖罚系统 (Reward & Penalty - Star System)**：
        * **奖励**：完成任务给予金色星星 (Gold Star)；画画或其他特殊表现给予紫色星星 (Purple Star)。
        * **惩罚**：情绪发脾气等记为惩罚，显示为空心或灰色形状（如正方形/三角形）。
        * **结算机制**：每周进行一次任务和情绪的汇总结算；家长可以随时手动给予奖励或修改星星总数。
    * **货币系统 (Currency System)**：
        * 兑换比例与人民币概念对应，为 1:10 (例如 1元=10金币)。
        * 可通过完成画作（售卖）等方式赚取货币。
        * 货币用途：可用于购买消费品（见消费系统），或用于“抵扣/消除”情绪惩罚记录。

## 业务功能模块 (Business Modules)

1. **任务系统 (Task System)**
    * 支持创建日常任务 (Daily Tasks) 和重复任务 (Repeating Tasks)。
    * 可以设定任务的开始时间和结束时间。
    * 完成任务后自动给予对应的奖励（金色星星），未完成则不给予奖励。

2. **情绪系统 (Emotion System)**
    * 重点用于记录“生气”、“发脾气”等突发状况。
    * 发生情绪失控时，由家长或孩子自己标记一次“生气惩罚”（记录灰色的特定形状）。

3. **画廊与售卖系统 (Gallery & Art Sales)**
    * **画册 (Albums)**：允许把图片归类为您自己的画册。
    * **动态 UI 呈现**：在画廊内，画册仿照“艺术展”风格，以2-3张图片混乱重叠作为封面，点击后展开详情列表。
    * **创作与上传**：支持设备拍照或从图库选中照片上传；上传时需选择所属画册并定义“售卖价格”。
    * **分享与交易**：发布后系统生成一个“带有图片 + 售卖价格 + 宣传标语”的分享图（海报），方便保存并发送给他人扫码。
    * **独立支付与外链**：他人扫码访问对应外链进行付费购买；没有账号的用户可通过系统快速注册一个专属访客账号。
    * **实时通知**：收到付款后，向设备推送 Web Push 提示，并在系统中永久记录购买者。

4. **日志系统 (Journal System)**
    * 提供多媒体记录：支持输入文字、上传图片，也可以发送语音。
    * 家长和孩子均可使用，记录各自视角的日记/日志（UI 上做出区分标记）。

5. **货币消费商城 (Shop/Consumption System)**
    * 一个内置的愿望与兑换商城。
    * 常见可购买商品预设：玩具、乐高积木、电子游戏时间、书籍、画册、冰淇淋、蛋糕等。
    * 孩子可随时查看余额，并根据自己的货币余额自主操作“购买”消费（家长后台确认与核销）。
