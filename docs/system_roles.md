# 系统角色与权限说明 (System Roles & Permissions)

本文件详细说明了系统内的不同角色定义、权限边界以及核心功能的交互逻辑。

---

## 1. 核心角色定义 (Core Persona Definitions)

系统划分为四类核心 Persona，每种角色在前端展示、路由访问及数据权限上具有明确边界。

### 1.1 Superadmin (超级管理员)
- **定义**：系统初始化时自动生成的最高权限账号。
- **管理权限**：
  - 进入管理控制台 (`/admin/console`) 管理全体成员与系统配置。
  - 唯一可将普通成员提升为 `Admin` 或撤销其管理权的角色。
- **操作特性**：
  - **HUD 限制**：在 HUD 弹窗中仅能为自己充值金币（Coins）和金星（Stars）。
  - **成长记录**：拥有为任一孩子（Child）角色记录成长数据（Telemetry）的权限。
  - **数据保护**：`isLocked: true`，不可被删除或降权。

### 1.2 Admin (管理员)
- **定义**：由 Superadmin 指定的具有管理权限的家长或长辈账号。
- **管理权限**：
  - 可访问管理控制台 (`/admin/console`) 进行成员管理与系统操作。
  - 无法对 Superadmin 账号进行修改、删除或降权。
- **操作特性**：
  - **HUD 限制**：在 HUD 弹窗中仅能为自己充值金币和金星。
  - **成长记录**：可为孩子角色记录成长数据。

### 1.3 Family Member (普通家庭成员 - 非管理员)
- **核心逻辑**：不论角色性质（Parent, Child, Grand 等），只要不具备 `Admin` 权限，其操作边界完全一致。
- **权限限制**：
  - **禁止访问**：绝对禁止访问 `/admin/console` 管理后台。
  - **成长记录**：无法记录成长数据，仅可查看统计图表（Stats）。
- **通用功能**：
  - **个人中心**：通过 `/member` 访问，查看收藏、订单及财务记录。
  - **画廊组件**：在主仪表盘（Bento Dashboard）拥有“我的画廊”组件。
  - **展示页面**：可拥有并管理个人的 Exhibition Page，支持公共访问。

### 1.4 Visitor (外部访客)
- **定义**：家庭外部的浏览者。
- **交互边界**：
  - 通过 `/visitor/login` 登录独立的访客中心。
  - 支持在公共展厅进行点赞（Likes）和下单。
  - 可管理个人配送地址以及访客端的余额。

## 2. 核心交互系统规则

### 2.1 任务系统 (Task Assignment)
系统鼓励家庭成员间相互协作：
- **P2P 指派**：支持跨角色（孩子给家长、孩子给孩子等）相互指派任务。
- **导航更新**：所有成员的侧边菜单均包含 **Assigned (被委派)** 与 **Assign To (委派记录)**。
- **奖励逻辑**：
  - **金星结算**：所有任务统一通过金星（Stars）结算，取消金币奖励。
  - **资产动态转移**：任务奖励来源于指派人。创建任务时会预冻结指派人的金星，任务完成后转移至执行人。若指派人金星不足，则不允许创建。

### 2.2 资产、兑换与结算 (Economy & Shop)
资产流转遵循以下物理规则：
- **资产兑换 (兑换入口位于 /admin/shop)**：
  - 支持“金星换金币”与“金币换金星”，比例按后台配置执行。
- **消费限制**：购买商品或服务只能使用 **金币 (Coins)**。
- **线下提现**：
  - 金币与法币比例为 1:1。
  - 提供金币提现入口，提交申请时需确认已完成线下真实资金交付。

### 2.3 成长与统计 (Growth Telemetry)
- **特定性**：系统中仅 **孩子 (Child)** 角色拥有成长记录和统计数据。
- **权限细分**：
  - `Admin`/`Superadmin`：可录入数据（身高、体重等）。
  - `Member`：仅支持数据可视化查看。

### 2.4 访客互动激励 (Visitor Engagement)
- **点赞奖励**：访客对作品每点一次 Likes，创作者获得 1 颗金星。仅首次点赞有效。
- **订单奖励**：访客订单完成处理后，创作者除了获得金币外，系统额外奖励 10 颗金星（奖励额度可在设置中调整）。

## 3. 架构与逻辑实现

### 3.1 路由架构 (Routing Architecture)

| 路由路径 | 适用角色 | 说明 |
| :--- | :--- | :--- |
| `/admin/*` | Superadmin / Admin | 系统管理、后台操作边界。 |
| `/member/*` | 任何已登录成员 | 家庭成员个人页及前台功能入口。 |
| `/visitor/*` | Visitor | 独立的访客认证与资产管理。 |
| `/u/[slug]` | 所有 | 公共展示主页，支持不同身份的无缝交互。 |

### 3.2 权限控制实现
- **RBAC 映射**：
  - `SUPERADMIN/ADMIN` -> `isAdmin: true`
  - `USER` -> `isAdmin: false`
- **AuthGate 模式**：
  - `FAMILY`: 仅限已登录家庭角色。
  - `VISITOR`: 仅限访客身份。
  - `BOTH`: 支持双端认证访问。
- **数据安全**：关键账号通过 `isLocked` 字段锁定，API 层强制拦截针对此类账号的删除请求。
